XSS Test Cases for Collaboration Editor

These test cases can be used to verify that the XSS protection is working:

1. Script Tag Injection

<script>alert('XSS')</script>

2. Image Tag with onerror Event

<img src="x" onerror="alert('XSS')">

3. Iframe with JavaScript URL

<iframe src="javascript:alert('XSS')"></iframe>

4. SVG with Script

<svg onload="alert('XSS')"></svg>

5. Input with onfocus Event

<input onfocus="alert('XSS')" autofocus>

6. Link with JavaScript URL

<a href="javascript:alert('XSS')">Click me</a>

7. Style Tag with CSS Expression

<style>body{background:expression(alert('XSS'))}</style>

8. Div with onclick Event

<div onclick="alert('XSS')">Click me</div>

9. Meta Tag with Refresh

<meta http-equiv="refresh" content="0;url=javascript:alert('XSS')">

10. Object Tag with Data URL

<object data="javascript:alert('XSS')"></object>

Expected Behavior

After implementing DOMPurify sanitization:

  • All script tags should be completely removed
  • All event handlers (onerror, onload, onclick, etc.) should be stripped
  • JavaScript URLs should be blocked
  • Only safe HTML tags (h1, h2, h3, p, strong, em, ul, li, br) should be allowed
  • No attributes should be allowed except for the safe ones configured

Testing Steps

  1. Start the application
  2. Navigate to the collaboration editor
  3. Try pasting each test case into the document editor
  4. Switch to the Preview tab
  5. Verify that no JavaScript executes and no alerts appear
  6. Check the browser's developer tools to ensure malicious code is stripped
